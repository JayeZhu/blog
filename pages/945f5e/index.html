tcp 三次握手

首先，客户端和服务端都处于close状态，客户端手动解除close状态，服务端被动进入 listen 状态

客户端发出其第一次握手，向服务端确认自己的发送能力，发送内容包括 syn 报文和初始化序列号 ISN，其中 syn = 1，seq = x

服务端收到后，确认客户端可以发送，接着向客户端确认服务端的发送能力，发送内容有 syn = 1，初始化 seq = y，ACK = 1，ack = x + 1

客户端收到后，确认了客户端的发送
